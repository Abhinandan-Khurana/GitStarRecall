# GitStarRecall - Landing Page Content Checklist

Use this checklist to ensure the landing page covers all required content and messaging.

---

## 1) Core Sections
- Hero section with product name, one-line value prop, and primary CTA.
- Demo video section (embedded YouTube/Vimeo or thumbnail + play).
- Security and local-first highlights.
- How it works (3-5 steps).
- Supported providers (remote + local).
- FAQ or small details section for power users.
- Footer with links (privacy, GitHub, docs).

## 2) Messaging Requirements
- Clear statement: data is stored locally by default.
- Clear statement: external LLM usage is opt-in.
- Mention SQLite WASM + `sqlite-vec-wasm` local search.
- Mention GitHub OAuth and private repo support via token.
- Mention session-based chat for follow-up queries.

## 3) CTA Requirements
- Primary CTA: "Connect GitHub" or "Start Search".
- Secondary CTA: "Watch Demo".
- CTA routes to login and then usage page.

## 4) Visual Requirements
- Demo video is visible above the fold or in the first scroll.
- Security/local-first messaging is visually emphasized (badges or callouts).
- UI shows a preview of results or session list.

---

## 5) Sample Copy (Starter)
- Headline: "GitStarRecall - find your starred repos by memory, not by name."
- Sub-headline extension: "Ask for tailored recommendations from your own stars based on your exact use case."
- Subhead: "Private by default. Local-first RAG that turns your GitHub stars into instant, searchable knowledge."
- Security callout: "Your data stays on your device. External LLMs are opt-in only."
- Local-first callout: "SQLite WASM + sqlite-vec-wasm means fast semantic search without sending your code anywhere."
- CTA: "Connect GitHub" / "Watch Demo"

---

## Completion Rule
Landing page is complete only when all checklist items are satisfied.
